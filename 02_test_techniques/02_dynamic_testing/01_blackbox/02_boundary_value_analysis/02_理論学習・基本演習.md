## 🎯 境界値分析の理論

### 📖 **境界値の定義と種類**

#### **境界値とは**
入力や出力の**有効範囲と無効範囲の境目付近の値**のことです。

#### **境界値の分類**

**1. 最小境界値（下限境界）**
- 有効範囲の最小値とその前後の値
- 例: 年齢0-120歳 → -1, 0, 1

**2. 最大境界値（上限境界）**
- 有効範囲の最大値とその前後の値
- 例: 年齢0-120歳 → 119, 120, 121

**3. 内部境界値**
- 有効範囲内での区分境界
- 例: 料金体系の年齢区分 → 12歳, 13歳, 17歳, 18歳

### 🔢 **境界値テストの種類**

#### **2点境界値テスト（Two-Point Boundary Value Testing）**
境界値の**両側の値**をテストする手法

```
仕様: 1-100の範囲で入力可能
2点境界値: 0, 1, 100, 101
```

**特徴**
- ✅ 効率的（テストケース数が少ない）
- ✅ 基本的な境界エラーを検出
- ❌ 境界値自体のバグは検出できない場合がある

#### **3点境界値テスト（Three-Point Boundary Value Testing）**
境界値の**両側 + 境界値そのもの**をテストする手法

```
仕様: 1-100の範囲で入力可能
3点境界値: 0, 1, 2, 99, 100, 101
```

**特徴**
- ✅ より確実な境界エラー検出
- ✅ 境界値での処理も検証
- ❌ テストケース数が増加

#### **使い分けの指針**
- **2点境界値**: 効率重視、リスクが低い機能
- **3点境界値**: 品質重視、クリティカルな機能

## 🔍 境界値特定の手順

### 📋 **Step 1: 同値分割の実行**

#### **同値分割で入力を分類**
```
例: パスワード文字数（8-20文字）

同値クラス:
1. 無効値（8文字未満）
2. 有効値（8-20文字）  
3. 無効値（20文字超過）
```

### 📋 **Step 2: 境界の特定**

#### **各同値クラスの境界を特定**
```
同値クラス境界:
- 8文字未満 ｜ 8-20文字 の境界 → 7-8文字
- 8-20文字 ｜ 20文字超過 の境界 → 20-21文字
```

### 📋 **Step 3: 境界値の抽出**

#### **2点境界値の場合**
```
下限境界: 7文字, 8文字
上限境界: 20文字, 21文字
```

#### **3点境界値の場合**
```
下限境界: 7文字, 8文字, 9文字
上限境界: 19文字, 20文字, 21文字
```

### 📋 **Step 4: テストケースの作成**

#### **境界値テストケース例**
| テストケース | 入力値 | 期待結果 | 境界種別 |
|-------------|--------|----------|----------|
| TC001 | 7文字 | エラー | 下限境界-1 |
| TC002 | 8文字 | OK | 下限境界 |
| TC003 | 9文字 | OK | 下限境界+1 |
| TC004 | 19文字 | OK | 上限境界-1 |
| TC005 | 20文字 | OK | 上限境界 |
| TC006 | 21文字 | エラー | 上限境界+1 |

## 🔧 境界値分析の実践ポイント

### 💡 **効果的な境界値特定のコツ**

#### **1. 仕様書の読み方**
```
✅ 数値範囲の記述を注意深く読む
✅ 「以上」「以下」「未満」「超過」の違いを確認
✅ 暗黙の制約も考慮する
```

#### **2. データ型を考慮**
```
整数: 1つずつ増減
小数: 最小単位で増減
文字列: 1文字ずつ増減
日付: 1日ずつ増減
```

#### **3. 実装を考慮した境界値**
```
配列インデックス: 0, 1, length-1, length
ループ回数: 0回, 1回, 最大回数
メモリ制限: 使用可能量の境界
```

### ⚠️ **よくある間違いと注意点**

#### **境界値の取り間違い**
```
❌ 1-100の範囲で 0, 1, 99, 100 とする
✅ 1-100の範囲で 0, 1, 100, 101 とする
```

#### **データ型の考慮不足**
```
❌ 小数で 1.0, 2.0 とする（1つ刻み）
✅ 小数で 1.0, 1.1 とする（最小単位刻み）
```

#### **同値分割との連携不足**
```
❌ 境界値だけを抽出
✅ 同値分割と組み合わせて設計
```
