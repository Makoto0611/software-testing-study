## 🛠️ 演習1: 文字種制限のある入力（パスワード）

### 📋 演習仕様
```
パスワード入力検証
制約:
- 長さ: 8-16文字
- 必須: 英大文字、英小文字、数字を各1文字以上
- 許可文字種: 英数字のみ（記号不可）
- 禁止パターン: 連続3文字以上の同一文字（例: "aaa", "111"）

エラーメッセージ:
- 長さ不足/超過: 「8-16文字で入力してください」
- 文字種不足: 「大文字、小文字、数字を含めてください」
- 記号含む: 「英数字のみ使用できます」
- 連続文字: 「同じ文字を3回以上連続できません」
```

### 📝 課題
1. 有効・無効同値クラスを列挙
2. 各クラスの代表値を選択
3. テストケース表を作成

### ✅ 解答例
```
有効同値クラス:
EC1: 8-16文字 + 英大小数字各1以上 + 記号なし + 連続なし
  代表値: "Password123"

無効同値クラス:
EC2: 8文字未満
  代表値: "Pass1"
EC3: 16文字超過
  代表値: "Password12345678"
EC4: 大文字なし
  代表値: "password123"
EC5: 小文字なし
  代表値: "PASSWORD123"
EC6: 数字なし
  代表値: "PasswordABC"
EC7: 記号含む
  代表値: "Pass@word123"
EC8: 連続3文字
  代表値: "Passsword123"
```

**ポイント**: 複数の制約がある場合、1つずつ違反する無効クラスを作成

## 🛠️ 演習2: 金額計算（段階的割引）

### 📋 演習仕様
```
購入金額別の会員ポイント付与率
条件:
- 1,000円未満: 1%
- 1,000-4,999円: 3%
- 5,000-9,999円: 5%
- 10,000-29,999円: 8%
- 30,000円以上: 10%

会員ランクによる追加ボーナス:
- 通常会員: +0%
- シルバー会員: +1%
- ゴールド会員: +2%

制約:
- 金額は0以上
- 整数のみ
```

### 📝 課題
この問題は「金額範囲」と「会員ランク」の2つの条件があります。
1. 金額範囲の同値クラスを特定
2. 会員ランクの同値クラスを特定
3. 基本的なテストケースを作成（組み合わせは考えなくてOK）

### ✅ 解答例
```
金額範囲の同値クラス:
EC1: 0-999円 → 1%
EC2: 1000-4999円 → 3%
EC3: 5000-9999円 → 5%
EC4: 10000-29999円 → 8%
EC5: 30000円以上 → 10%
EC6: 負の値 → エラー
EC7: 非整数 → エラー

会員ランクの同値クラス:
EC8: 通常会員 → +0%
EC9: シルバー会員 → +1%
EC10: ゴールド会員 → +2%
EC11: 不正なランク → エラー

基本テストケース:
TC001: 500円 + 通常 → 1%
TC002: 3000円 + 通常 → 3%
TC003: 7000円 + シルバー → 5%+1%=6%
TC004: 20000円 + ゴールド → 8%+2%=10%
TC005: -100円 → エラー
```

**ポイント**: 2つの条件がある場合、まず個別に同値分割してから組み合わせを考える

## 🛠️ 演習3: 日付範囲入力

### 📋 演習仕様
```
イベント期間設定
制約:
- 開始日: 今日以降
- 終了日: 開始日以降
- 期間: 最大90日間
- 営業日のみ（土日祝日は不可）
- 6ヶ月先まで予約可能

エラー:
- 開始日が今日より前: 「過去の日付は指定できません」
- 終了日が開始日より前: 「終了日は開始日以降にしてください」
- 期間91日以上: 「期間は90日以内にしてください」
- 土日祝日: 「営業日を指定してください」
- 6ヶ月超過: 「6ヶ月先まで予約可能です」
```

### 📝 課題
日付の依存関係に注意して同値クラスを設計してください。

### ✅ 解答例（簡略版）
```
有効同値クラス:
EC1: 開始日=明日（平日）, 終了日=開始+30日（平日）, 期間≤90日
  代表値: start=2025-10-15, end=2025-11-14

無効同値クラス:
EC2: 開始日が過去
  代表値: start=2025-10-13
EC3: 終了日が開始日より前
  代表値: start=2025-10-20, end=2025-10-15
EC4: 期間91日以上
  代表値: start=2025-10-15, end=2026-01-15（92日）
EC5: 開始日が土日
  代表値: start=2025-10-18（土曜日）
EC6: 6ヶ月超過
  代表値: start=2026-05-01
```

**ポイント**: 日付の依存関係は複雑なので、1つずつ制約を確認

## 🛠️ 演習4: 列挙型（複数選択）

### 📋 演習仕様
```
アンケート回答システム
質問: 「興味のある分野を選択してください（複数選択可）」
選択肢: プログラミング、デザイン、マーケティング、ビジネス、その他

制約:
- 最低1つ、最大3つまで選択
- 重複選択は不可
- 存在しない選択肢は不可

エラー:
- 未選択: 「最低1つ選択してください」
- 4つ以上: 「最大3つまで選択できます」
- 重複: 「重複して選択できません」
- 不正な選択肢: 「無効な選択肢です」
```

### 📝 課題
複数選択の同値クラスを設計してください。

### ✅ 解答例
```
有効同値クラス:
EC1: 1つ選択
  代表値: ["プログラミング"]
EC2: 2つ選択
  代表値: ["プログラミング", "デザイン"]
EC3: 3つ選択
  代表値: ["プログラミング", "デザイン", "マーケティング"]

無効同値クラス:
EC4: 未選択
  代表値: []
EC5: 4つ以上選択
  代表値: ["プログラミング", "デザイン", "マーケティング", "ビジネス"]
EC6: 重複選択
  代表値: ["プログラミング", "プログラミング"]
EC7: 不正な選択肢
  代表値: ["プログラミング", "無効な選択肢"]
```

**ポイント**: 複数選択の場合、「個数」と「内容の妥当性」を分けて考える

## 🛠️ 演習5: ファイルアップロード

### 📋 演習仕様
```
画像アップロード機能
制約:
- ファイルサイズ: 100KB以上10MB以下
- ファイル形式: jpg, png, gif のみ
- 画像サイズ: 横幅 200px以上2000px以下
- ファイル名: 英数字とハイフン、アンダースコアのみ
- 最大文字数: 50文字

エラーメッセージ:
- サイズ小: 「ファイルサイズは100KB以上必要です」
- サイズ大: 「ファイルサイズは10MB以下にしてください」
- 形式違反: 「jpg, png, gif形式のみ対応しています」
- 画像サイズ小: 「横幅は200px以上必要です」
- 画像サイズ大: 「横幅は2000px以下にしてください」
- ファイル名不正: 「ファイル名に使用できない文字が含まれています」
- ファイル名長い: 「ファイル名は50文字以内にしてください」
```

### 📝 課題
複数の観点（サイズ、形式、画像サイズ、ファイル名）から同値クラスを設計してください。
